### Step-by-Step

1. User clicks "Get Started" on the landing page
2. Privy modal opens (X or email login options)
3. Privy handles the OAuth flow and returns a Privy access token
4. Frontend calls `POST /api/auth/privy/login` with the Privy token
5. Backend validates the Privy token via Privy server SDK
6. If new user: creates account + auto-generates Base & Solana wallets + default settings
7. Backend issues a JWT (HS256, 7-day expiry)
8. JWT stored in localStorage, sent on every API request via `Authorization: Bearer` header

## Auth Methods

| Method | Purpose |
|--------|---------|
| X (Twitter) OAuth | Primary signup/login вҖ” connects X account for like monitoring |
| Telegram OAuth | TG-first signup/login via Privy вҖ” creates account without X |
| Email | Account recovery path |
| Privy re-auth | Fresh token required for wallet export |

---

## JWT Authentication

- **Algorithm:** HS256
- **Expiry:** 7 days
- **Payload:** `{ userId, xUsername, isPremium }`
- **Stored:** Frontend localStorage
- **Sent:** `Authorization: Bearer <token>` header on every request
- **Validation:** `authMiddleware` verifies JWT signature and extracts `req.user`

---

## Privy Re-Authentication

For wallet export specifically, Privy re-auth is required:

1. Frontend requests a fresh Privy access token
2. Token sent to `POST /api/wallet/export-key/request`
3. Backend verifies the Privy token is fresh (not reused)
4. HPKE encryption key exchange occurs
5. Encrypted private key returned to frontend
6. Frontend decrypts using HPKE (DHKEM\_P256 + HKDF\_SHA256 + CHACHA20\_POLY1305)

## Telegram Signup (TG-First)

Users can create accounts directly via Telegram OAuth through Privy, without needing an X account:

1. User clicks "Get Started" and selects Telegram login in the Privy modal
2. Privy handles the Telegram OAuth flow (CSP allows `https://auth.privy.io` in `script-src` and `https://oauth.telegram.org` in `connect-src`)
3. Frontend calls `POST /api/auth/privy/login` with the Privy token
4. Backend creates account using `telegramUsername` as the display name (no `xUsername`)
5. Auto-generates Base & Solana wallets + default settings
6. User can optionally connect their X account later via `POST /api/auth/connect/x` to enable like-to-buy

**Dashboard fallback:** TG-only users see their Telegram username throughout the dashboard (sidebar, welcome message) via the fallback chain: `xUsername || telegramUsername || 'user'`

---

## Telegram Connection (Existing Users)

Users with an existing account can link their Telegram for trade notifications:

1. User calls `POST /api/auth/connect/telegram` with their Telegram username
2. Backend generates a 6-character connect code
3. User opens the Telegram bot and sends `/connect <code>`
4. Bot verifies the code and links the Telegram chat ID to the user
5. Notifications now delivered via Telegram DM

**Guard:** If a user tries to connect an X account that already belongs to another user, the connection is rejected to prevent account merging conflicts.

## Rate Limiters

All auth endpoints have dedicated rate limits to prevent abuse:

| Endpoint | Window | Max Requests |
|----------|--------|-------------|
| General API | 60s | 30 (free) / 120 (premium) |
| Auth endpoints | 60s | 10 |
| Privy login | 60s | 10 |
| X OAuth connect | 60s | 5 |
