# Narrative Token Launch

CLIQI's AI-powered narrative engine autonomously detects emerging trends across ALL of X — politics, AI, sports, world events, culture, and crypto — and tokenizes them on Base via Bankr before they go mainstream. Creator fees are used to buy back ECHO, the narrative product token.

The narrative account also posts **trending news updates** (Watcher Guru-style) to X between token launches — keeping the audience engaged even when no token qualifies for launch.

---

## How It Works

```
X API (dedicated app)          ← Fetch trending tweets across ALL topics
        ↓
Grok / xAI (grok-3-mini)      ← Cluster & score emerging trends
        ↓
Bankr Deploy API               ← Launch token on Base (gasless)
        ↓
Telegram DM                    ← Notify premium users FIRST (1.5 min head start)
X Bot (@_Echoonbase)           ← Post launch to X (full CA shown)
        ↓
Users trade the token
        ↓
Bankr creator fees (WETH)      ← 57% of 1.2% swap fee
        ↓
Bankr Agent API                ← Claim fees (1/day per token) + accumulate + buyback at $100
  └── 100% WETH → ECHO token buyback
```

## Trend Detection

### Data Collection

A **dedicated X developer app** (separate from the like-polling app) searches for emerging trends across 8 mainstream categories:

- **Breaking news & viral moments** — "just announced", "this is huge", trending phrases
- **Politics & policy** — Trump, elections, sanctions, tariffs, trade wars
- **AI & tech** — AI breakthroughs, ChatGPT, robotics, quantum computing
- **Geopolitics & conflict** — wars, ceasefires, peace deals, military events
- **Economy & markets** — Fed decisions, inflation, interest rates, crashes, rallies
- **Pop culture & sports** — championships, viral moments, scandals
- **Science & space** — SpaceX, NASA, climate events, discoveries
- **Crypto** — only as one of many signals, not the primary focus

Tweets are fetched via X API v2 `GET /2/tweets/search/recent` with engagement metrics (`public_metrics`) and author data. The key insight: catch trends **before** crypto Twitter picks them up.

### Scoring (Grok)

Tweets are sent to Grok (`grok-3-mini`) acting as a **trend-spotter** which:

1. Clusters tweets by theme/trending topic
2. Scores each cluster on:

| Signal | Weight | Description |
|--------|--------|-------------|
| Unique accounts | 30% | Breadth of discussion |
| Engagement velocity | 25% | Likes + RTs per hour |
| Influencer overlap | 20% | Multiple KOLs on same theme |
| Freshness | 15% | Emerged in last 6 hours |
| No existing token | 10% | Not already launched elsewhere |

3. Returns the top trend with a FOMO-style summary, suggested token name, symbol, and description
4. Only narratives scoring **≥ 70** (configurable) with **≥ 10 unique accounts** qualify
5. Mainstream narratives (politics, AI, world events) are **prioritized** over crypto-native ones

### Quality Gates

- Maximum **1 launch per day** (configurable via `MAX_LAUNCHES_PER_DAY`)
- Same theme has a **7-day cooldown** (can't launch twice)
- **Recent themes blacklist** — the scorer receives the last 20 launched token names/symbols/narratives, and Grok is instructed to pick something completely different
- Engine can be **paused** via `NARRATIVE_PAUSED=true` env var
- Narrative lifespan is **manually controlled** by admin (`PATCH /api/narratives/:id/complete`)

## Token Launch (Bankr)

Tokens are deployed on **Base only** via Bankr's Deploy API:

```
POST https://api.bankr.bot/token-launches/deploy

{
  "tokenName": "AI Agents Meta",
  "tokenSymbol": "AIMETA",
  "description": "The AI agent narrative token launched by CLIQI"
}
```

- **Gasless** — Bankr sponsors all gas fees
- **No upfront cost** — 1.2% swap fee model
- **Creator revenue** — CLIQI earns 57% of swap fees (WETH), paid to the Bankr agent wallet
- Fees go to the Bankr agent wallet by default (no `feeRecipient` set), enabling automated claim + buyback

## Fee Buyback

The buyback engine runs as an **hourly SafeInterval** background job. All claimed WETH fees are used to buy back **ECHO**, the narrative product token.

### Per-Token Accumulation

Since all claimed fees land as WETH in a shared Bankr wallet, fees are attributed per-token using a **balance delta** approach:

1. Snapshot WETH balance **before** claiming
2. Claim fees for the token (max **1 claim per day per token** — Redis gate prevents Bankr API spam)
3. Snapshot WETH balance **after** claiming
4. `earned = after - before` — only this token's fees
5. Add `earned` to a Redis accumulator: `narrative:weth-accum:{tokenId}`
6. When accumulated WETH (at current ETH price) ≥ **$100** → execute buyback and reset accumulator

The claim date is set in Redis **before** calling Bankr, so if the call fails or returns 429, it won't retry until the next day.

### Accumulation Example

```
Day 1 — $BAGI
  wethBefore = 0.00, claim, wethAfter = 0.04
  earned = 0.04 WETH → Redis[BAGI] = 0.04 ($86) → below $100, skip

Day 2 — $BAGI
  wethBefore = 0.04, claim, wethAfter = 0.05
  earned = 0.01 WETH → Redis[BAGI] = 0.05 ($108) → OVER $100
  → buyback immediately, Redis[BAGI] reset to 0
```

### Buyback Allocation

| Destination | Allocation | Description |
|-------------|------------|-------------|
| ECHO token | 100% | All WETH fees → ECHO token buyback |

All swaps execute via Bankr's `/agent/prompt` → `/agent/job/{jobId}` async pattern:

```
"swap 0.050 WETH for <ECHO_TOKEN_ADDRESS> on base"   ← 100%
```

The ECHO token contract address is configured via the `ECHO_TOKEN_ADDRESS` env var.

### Multiple Active Tokens

Each active token is processed sequentially. The balance delta per claim ensures tokens don't contaminate each other's accumulators even when WETH from previous runs remains in the wallet.

## Distribution

### Premium Alpha: Telegram DM (sent FIRST — 1.5 min head start)

Premium users receive a Telegram DM **90 seconds before** the public X post — giving them early access to buy the narrative token before anyone else.

Notifications are **auto-enabled** when a user connects their Telegram account and can be toggled individually via `tgNotifyNarrativeLaunch`.

The TG notification includes:
- Token name, symbol, FOMO-style narrative summary, and score
- BaseScan link
- Inline **Buy** button with preset amounts: **$5 / $10 / $25 / $50 / $100**
- Cancel option to dismiss the buy prompt

The buy respects the user's configured **Base trading mode**:
- `USDC` mode → gasless USDC swap (direct)
- `ETH` mode → USD amount converted to ETH equivalent, then swapped

**Note:** News updates (see below) are **not** sent to Telegram — only token launches get TG notifications.

### Public: X (@_Echoonbase)

After the 90-second TG head start, the token is posted to X via the dedicated **@_Echoonbase** account with:
- Token symbol and name
- FOMO-style narrative summary
- Score
- BaseScan link

Tweets are hard-capped at **280 characters** (X API returns 403 for longer tweets). The narrative summary is trimmed to fit within the limit.

No @mentions or #hashtags are included in automated posts to avoid X platform restrictions.

## News Updates (Watcher Guru-style)

Between token launches, the **@_Echoonbase** account posts trending news updates to keep the audience engaged. This runs as a separate background job (`narrative-news`) every **4 hours**.

### How It Works

1. Fetch the same trending tweets used by the narrative engine
2. Send to Grok with a **viral news prompt** — punchy, dramatic, scroll-stopping style
3. Grok extracts the biggest trending headline across 8 categories (Politics, AI & Tech, War & Geopolitics, Economy & Markets, Sports & Culture, Science, Crypto, Breaking News)
4. Post to X with a category emoji and headline + summary

### Quality Gates

- Maximum **4 news posts per day**
- Minimum **3 hours between posts** (prevents deploy spam — Redis timestamp gate)
- **48-hour topic dedup** — same topic won't be posted twice within 48h (Redis set)
- Confidence threshold: Grok must score the headline ≥ 60
- Combined headline + summary must fit within **280 characters**
- No @mentions or #hashtags

### News vs Token Launches

| | News Updates | Token Launches |
|---|---|---|
| Frequency | Up to 4/day | Max 1/day |
| Telegram | No | Yes (1.5 min before X) |
| X post | Yes (@_Echoonbase) | Yes (@_Echoonbase) |
| Token created | No | Yes (Bankr) |

## Revenue Model

```
Token launched via Bankr
        ↓
Users trade on DEX (1.2% swap fee)
        ↓
Fee split:
  ├── 57% → CLIQI (creator)     ← WETH to Bankr agent wallet
  ├── 36.1% → Bankr
  ├── 1.9% → Ecosystem
  └── 5% → Protocol (Doppler)
        ↓
CLIQI's 57% → 100% ECHO token buyback
```

All creator WETH fees are used to buy back **ECHO** — the narrative product token on Base. This creates continuous buy pressure on ECHO as long as narrative tokens are being traded.
