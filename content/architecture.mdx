## Database Schema (Key Models)

```prisma
User          # id, xUserId, xUsername, email, isPremium, premiumUntil, wallets[], trades[]
Wallet        # userId, chain (SOLANA|BASE), address, privyWalletId, walletType (EOA|SMART_WALLET)
Trade         # userId, chain, tokenAddress, action (BUY|SELL), amountIn/Out, amountInUsd,
              # txHash, txStatus (PENDING|SUBMITTED|CONFIRMED|FAILED|REJECTED), platformFee
UserSettings  # userId, solUsdBuyAmount, baseUsdcBuyAmount, maxSlippageBps, dailyLimitUsd,
              # autoSellEnabled, takeProfitPct, stopLossPct, tgNotificationsEnabled,
              # tgNotifyNarrativeLaunch
PremiumPayment # userId, paymentChain, amountPaid, amountUsd, txHash, status, startDate, endDate
PollingState   # userId, lastTweetId, lastPolledAt, pollErrors, isRateLimited
TokenCache     # address+chain (composite), symbol, name, riskScore, liquidityUsd, isHoneypot
NarrativeToken # narrative, tokenAddress, tokenSymbol, chain (BASE), score, status, totalBuybackUsd
NarrativeBuyback # narrativeTokenId, feeSourceUsd, tokenBuyUsd, cliqiBuyUsd, txHashes, status
```

## Middleware Stack (order matters)

```
1. helmet()                  # Security headers
2. CORS (whitelist)          # Origin enforcement
3. https-enforcer            # Force HTTPS in production
4. scanner-shield            # Block probe requests
5. morgan (HTTP logging)     # Request logging
6. express.json()            # Body parsing
7. sentry.requestHandler()   # Sentry request context
8. Rate limiters             # Per-route rate limiting
9. authMiddleware            # JWT verification (protected routes)
10. Route handlers           # Business logic
11. sentry.errorHandler()    # Sentry error capture
12. Global error handler     # Catch-all вҶ’ admin TG notification
```
